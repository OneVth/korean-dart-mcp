## 대원칙

범위(시장·업종·취향)는 사용자 고유 선택이다.
사용자가 명시하지 않았으면 **추측하지 말고 반드시 물어라.**
"이 사용자는 KOSPI를 원할 것" 같은 임의 판단 금지.
큰 결정은 사용자가 한다 (사경인 7부).
스캔의 killer 멈춤에서도 선택(전부/골라서/명단)은 사용자가 한다 — choice 임의 결정 금지.

## 1단계 — 취향 확인

`sagyeongin_user_preference(action="get")` 조회.

- 취향이 있으면 → 그 취향대로 진행할지 사용자에게 확인
- 취향이 없으면 → "전체 스캔 / 시장(KOSPI·KOSDAQ) / 특정 업종 중 어떻게 시작하시겠습니까?" 질문
- 임의로 범위를 정하지 않는다

## 2단계 — 범위 확정 (월권 방지 핵심)

`sagyeongin_scan_preview(...)` 로 콜 0 견적 먼저.

**스캔 규모 초과 시 (추정 콜 > 10,000):**

- Claude가 멋대로 좁히지 않는다
- 견적(추정 콜 수, universe 크기)을 사용자에게 그대로 보여준다
- "업종으로 좁힐지 / KOSPI만 / KOSDAQ만 / 그대로 진행할지" **선택을 사용자에게 돌린다**
- 여기서 멈추고 사용자 답을 기다린다

**적정 규모(≤ 10,000) 시:**

- "약 N콜 예상합니다. 진행할까요?" 확인 한 번

## 3단계 — 스캔 실행 + killer 멈춤 (ADR-0032)

`sagyeongin_scan_execute(...)`.

스캔은 killer(7부 A 탈락 검사) 통과 후 **항상 멈춘다.** killer 통과 = 1차 선별 — 사용자가 보고 결정한다.

**멈춤 응답:**

- N ≤ 10 → 종목명까지 표시됨
- N > 10 → 개수만 ("killer 통과 N개")

**선택지를 사용자에게 제시하고 답을 기다린다 (멋대로 진행 금지):**

1. **전부 분석** — 통과 전체를 srim 적정가 분석.
   `scan_execute(resume_from=scan_id, choice="all")`
2. **골라서 분석** — 사용자가 종목 지정 → 그것만 srim.
   `scan_execute(resume_from=scan_id, choice="selected", selected_corp_codes=[...])`
   - N > 10이라 종목명이 안 보이면, 사용자가 고르려 할 때 명단을 펼쳐 보여준다.
3. **명단으로 충분** — srim 생략, killer 통과 명단(가나다순)이 결과.
   `scan_execute(resume_from=scan_id, choice="list_only")`

선택을 사용자에게서 받아 위 인자로 재호출한다. **choice를 임의로 정하지 않는다.**

## 3-2단계 — srim 결과 (choice=all/selected 후)

전부/골라서 선택 시 srim 패스 완료 → candidates (composite_score 정렬, 종목명, ROE·K) 제시.
"어느 종목을 깊게 분석할까요? 또는 범위를 바꿔 재스캔하시겠습니까?" 질문.

## 4단계 — 단일 종목 분석

`sagyeongin_watchlist_check(corp_input=...)` 로 통합 점검.
필요 시 개별 도구 추가 호출:

- `sagyeongin_srim` — 적정가·안전마진 (7부 D)
- `sagyeongin_killer_check` — 탈락 여부 (7부 A)
- `sagyeongin_cashflow_check` — 영업CF 품질 (7부 B)
- `sagyeongin_capex_signal` — 성장 투자 신호 (7부 C)
- `sagyeongin_dividend_check` — 배당 성향 (7부 E)
- `sagyeongin_insider_signal` — 내부자 매매 신호

7부 관점 종합 해석 제공.

## 업종 코드 매핑 (KSIC 2자리 대분류)

> **주의**: `induty_code`는 5자리로 저장되며 `startsWith` 매칭.
> 2자리 prefix로 대분류 전체를 지정한다.
>
> 잘못된 예: `"064"` (3자리 — 매칭 안 됨)
> 올바른 예: `"26"` (전자부품·컴퓨터 전체)
>
> DART DB는 KSIC 9차/10차 혼재 (ADR-0026). 같은 2자리여도 개정 차수에 따라 의미가 다를 수 있음.
