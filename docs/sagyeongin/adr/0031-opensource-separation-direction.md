# 0031 - 오픈소스 공개 시 분리 구조 방향

- 상태: Proposed
- 결정일: 2026-06-02
- 결정자: 사용자 + Claude
- 관계: β-i 가드(src/lib 무변경)가 분리의 토대. 실행은 MVP 검증 후 별 ADR.

## 컨텍스트

이 프로젝트는 **오픈소스 공개 목적**으로 구축 중이다.

현재 한 저장소에 세 층이 섞여 있다:
1. **원본 korean-dart-mcp** (chrisryugj, MIT) — DART 데이터 도구 + npx 설치 인프라
2. **사경인 프레임워크** (7부 도구 15개, src/tools/sagyeongin) — build에 포함돼 npm 배포됨
3. **funnel skill** (.claude/skills/) — npm files 목록에 없어 배포 제외

현 배포 상태의 문제:
- `npx korean-dart-mcp` 설치 시 (1)+(2)는 오나 (3) skill 누락 → 대화형 funnel 흐름 빠짐
- 원본/사경인 경계가 코드에 섞여 라이선스·크레딧·원본 업데이트 추적이 불명확
- "남이 이걸 어떻게 설치/fork/기여하나"가 공개 전 풀려야 할 문제

## 고려한 옵션

- **(가) 통합 유지 + skill 배포 포함**: package.json files에 .claude 추가, 한 패키지로. 빠름. 그러나 원본/사경인 한 덩어리 — 경계 흐림, 원본 업데이트 머지 부담, 라이선스 모호.
- **(나) 분리**: 원본 korean-dart-mcp를 의존성(dependency)으로, 사경인 도구+skill을 별도 패키지/저장소로. 경계 명확, 원본 업데이트 버전으로 수령, 기여·라이선스 정직. 단 작업 큼(src/lib vs src/tools/sagyeongin 빌드 분리 — ADR급).
- **(다) 독립 fork**: 원본에서 갈라져 전부 내 것으로. 당장 편하나 원본 개선·DART API 변경 대응 전부 자가 부담, 크레딧 인상 나쁨.

## 결정

**(나) 분리 방향 채택. 단 실행은 MVP 검증 후.**

- 공개 시 구조: 원본 korean-dart-mcp = 의존성, 사경인 확장(7부 도구 + funnel skill) = 별도 패키지. 독립 fork 아님.
- 공개 시점: MVP 실사용 검증 완료 후 (날짜 미확정 — "쓸 만하다" 확인이 신호).
- 실행 순서: MVP 검증 → UX 디테일 + 분리 리팩토링 + 문서 → 공개.
- 본 ADR은 **방향 기록(Proposed)**. 분리 난이도(src/tools/sagyeongin이 src/lib에 의존하는 정도)는 검증 후 회수로 가늠, 실제 분리는 그때 별 ADR(Accepted)로.

## 근거

- **라이선스·크레딧 명확성**: 원본 MIT(chrisryugj) ↔ 사경인 확장(본인) 경계가 의존성으로 또렷. 오픈소스 신뢰의 토대. (가)는 한 덩어리라 모호, (다)는 원작 크레딧 인상 나쁨.
- **원본 업데이트 추적**: 분리면 의존성 버전 갱신으로 수령. (가)/(다)는 머지 충돌·자가 부담. DART API 변경 이력(예: fnlttSinglAcnt 계열) 고려 시 원본 유지보수를 원작자에 맡기는 게 지속가능.
- **β-i 가드가 토대**: 그간 src/lib(원본) 무변경 규율을 지켜옴 → 원본과 사경인 경계가 이미 코드에 어느 정도 서 있음. 분리가 처음부터 불가능한 얽힘은 아닐 가능성(검증 후 확인).
- **검증 전 분리 보류 근거**: 본인도 use case 굳히는 중. 검증 안 된 구조를 리팩토링하면 시간 낭비 + 검증 중 구조 변동 위험. "MVP 됐다 ≠ 공개 준비됐다" — 공개는 UX·설치·문서 문턱이 별도.
- **skill 배포 누락은 별도 기록**: 현 npm files에 .claude 없음 → "npx 설치는 skill 미포함, 로컬 clone 필요"를 공개 전 해소 대상으로 백로그/README에 명시.

## 결과

- 오픈소스 공개 시 구조 방향이 문서화됨 (Proposed — 합의된 방향, 실행 결정 아님).
- 분리 리팩토링은 본 ADR 범위 밖. 실행 ADR은 MVP 검증 후 별도 신설.
- skill npm 배포 누락(npm files에 .claude 미포함) 인지 기록 — 공개 전 해소 대상.
